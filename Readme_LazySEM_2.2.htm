1.1      How to use?

Run LazySEM by either executing the .exe file directly, or by installing Autohotkey and running the .ahk source code. Autohotkey can be downloaded here:

https://autohotkey.com/download/

Regardles of how you run LazySEM, an icon should appear in the toolbar and you’re ready to execute any of the hotkeys described below.

If you have several keyboard languages installed, be warned that pressing CONTROL+SHIFT may change your keyboard language. In that case, many hotkeys won’t behave like you expect them to, because you are actually typing different characters. The best thing to do is to turn off the windows hotkey for changing your keyboard layout. In Windows 7:

Control Panel -> Regional and Language Options -> Keyboards and Languages

Press the Change Keyboards button, go to Advanced Key Settings tab

Here you can choose the "Between input languages" item, and press Change Key Sequence button. Change to "Not Assigned".

You can also try pasting this in Start Menu/Run and pressing enter:

powershell -Command Set-ItemProperty -Path 'HKCU:\Keyboard Layout\Toggle' -Name HotKey -Value 3

 

1.2      Mplus Automation Hotkeys

1.2.1     The emergency escape hatch.              [CONTROL][ALT][Q]

Effect: Aborts and reloads the program.

Method: Hotkeys are not entirely foolproof, although I try to program them in a way that works on all systems. Should the code ever run amok and do things you don't want it to do, press this hotkey. The program will stop what it's doing and restart. You should clean up whatever mess it leaves (hint: MPlus and Excel, the programs most used by LazySEM, both keep a long Undo history, so you can usually press [CONTROL][Z], if the program deleted any of your work).

1.2.2     Write syntax for all possible correlations [CONTROL][SHIFT][C]

Effect: Given a space-separated list of variable names, creates syntax for all possible correlations between those variables. One correlation per line, so it is easy to create parameter labels.

Method: Highlight a space-separated list of variable names (without ; at the end of the line), and fire the hotkey.

1.2.3     Repeat sections of syntax for multiple waves etc [CONTROL][SHIFT][R]

Effect: The hotkey repeats the selected section of syntax as many times as specified by the user, updating numbers reflecting (e.g.) measurement wave.

Method: Use placeholders where you want the script to insert numbers reflecting the measurement wave. The following placeholders are recognized:

{1:6}, {num:num}

The section of syntax should always contain one command indicating the numbers to iterate over. In this case, the syntax would be repeated six times, for the numbers 1, 2, 3, 4, 5, 6. {3:5} would iterate over the numbers 3, 4, and 5.

{R}

This placeholder is replaced with the number

{R+1}, {R-1}, {R+/- num}

This placeholder is replaced with the number, plus or minus an integer.

 

Example:

Input: de{R+1} ON mcon{1:3};

Output:

de2 ON mcon1;
de3 ON mcon2;
de4 ON mcon3;

1.2.4     Label parameters [CONTROL][SHIFT][L]

Effect: Labels your MPlus parameters if they are unlabeled. If they are already labeled, removes numbers from the labels in an attempt to constrain labels to be equal. If they are already constrained, removes labels entirely. Skips empty lines and lines that do not look like valid MPlus commands.

Method: Select a block of valid commands in the MODEL section of your MPlus input file, or select your entire MODEL section. Fire the hotkey. If the selection does not contain parameter labels, each parameter now receives its own parameter label, based on the variable names and the type of command (means, variances, ON, WITH, and BY supported). The code skips lines it does not recognize as a command (those that do not contain ";"). If the selection does contain parameter labels, the script removes numbers from those labels, in order to constrain them to be equal.

1.2.5     Comment selection [CONTROL][SHIFT][!]

Effect: Comments the current selection. Works in MPlus, RStudio, and TexStudio.

Method: Select a number of rows of syntax and fire the hotkey. The rows will be commented out with the appropriate comment character of the editor you are using.

1.2.6     Extract MPlus model fit indices [CONTROL][SHIFT][Q]

Effect: Extracts model fit indices from an active output file and pastes them into an open Excel file. Runs an active input file first if no output file is active.

Method: Open an MPlus in/output file and an Excel file. Place the cursor on the Excel cell where you would like to start pasting the results. Make the MPlus file active and fire the hotkey.

Note: This hotkey searches through the output to obtain its results. If there are no model fits in your output, it may give an error. Use [CONTROL][ALT][Q] to restart the program, and inspect your output for errors.

Sometimes, the search doesn't work, and it will not return any fit indices. If this happens, run the hotkey again on the output file. This is a known bug with unknown sollution.

 

1.2.7     Create a script for series of Wald tests [CONTROL][SHIFT][W]

Effect: Creates a script for a series of Wald tests (to be executed by pressing [CONTROL][SHIFT][T].

Method: Select a block of syntax lines with labeled parameters you want to compare, and execute the hotkey. The hotkey will create a script for a series of Wald tests to compare these labeled parameters and append it at the end of your MPlus input file.

1.2.8     Conduct a series of Wald tests [CONTROL][SHIFT][T]

Effect: Runs a series of Wald tests in MPlus, and prints the results into an open Excel file.

Method: This function executes a series of Wald tests, one by one, and pasts the resulting Chi square tests to Excel. Normally, to do a Wald test, you insert the following syntax below your MODEL specification:

 

MODEL TEST:

0 = [parameter constraint 1]-[parameter constraint 2];

 

This hotkey allows you to run multiple Wald tests automatically, and grab a cup of coffee. The hotkey uses MPlus' sign for comments, "!" to comment out the list of Wald tests, and then uncomments and runs one line at a time. Therefore, each command for this script begins with a "!". The scripts can be automatically generated by selecting a couple of lines with parameter labels and pressing [CONTROL][SHIFT][W], or by hand. You need to know the following commands:

!Start wald <- Place this at the point where you want to begin your series of Wald tests.

!0 = [parameter constraint 1]-[parameter constraint 2]; <- This is a single Wald test.

!r <- R stands for Return, a fancy word for Enter. This introduces a line break in the Excel output.

!x <- X stands for Exit. This stops the series of Wald tests.

To do so, write the following syntax:

 

Example:

 

MODEL TEST:

!Start Wald

!0 = [parameter constraint 1]-[parameter constraint 2];

!0 = [parameter constraint 2]-[parameter constraint 3];

!0 = [parameter constraint 3]-[parameter constraint 4];

!r

!0 = [parameter constraint 4]-[parameter constraint 5];

!0 = [parameter constraint 5]-[parameter constraint 6];

!x

 

Note: This process may fail if MPlus gives an error while running, or performs an unexpected behavior. You may be able to help the script along, for example by running the MPlus input file. Alternatively, use [CONTROL][ALT][Q] to restart the program, and inspect your output for errors. If the script does not copy the results of the Wald tests to Excel, don’t worry: they are on the clipboard. Manually paste them to Excel after the script is done running.

 

1.2.9     Extract MPlus Results [CONTROL][SHIFT][E]

Effect: Extracts the Names, Estimates, Standard Errors and P-Values from a selected block of MPlus output. The results are pasted to Excel in the following format:

Parameter        Estimate (SE)   P-value

If you do not want the standard errors between brackets, activate the Excel window, make sure the extracted results are selected, and execute the hotkey again.

When running a multi-group model, it is possible to select the output from all groups. The results will be paneled by group when they are pasted in Excel.

Method: Open an empty Excel document. Open MPlus output file. Select a block of results in an output file, and [CONTROL][SHIFT][E].

Notes: Make sure to select the entire block of text; don't skip the empty space on the first line or select the beginning of the next line.

To plug the MPlus results into a Visio drawing, see Plug results into Visio drawing.

1.2.10   Add significance asterisks [CONTROL][SHIFT][*]

Effect: Adds significance asterisks to parameter estimates.

Method: Open an Excel document. Select two columns of data: parameter estimates (left column) and p-values (right column). The output generated by Extract MPlus Results is already in this format. Press [CONTROL][SHIFT][*], significance asterisks corresponding to the p-value will be added to the parameter estimate.

Note: P-values smaller than .08 and greater than .05 are indicated with a cross. EQUAL TO or smaller than .05 are indicated with one asterisk. Smaller than .01 are indicated with two asterisks, and smaller than/equal to .001 are indicated with three asterisks, because some statistical programs round <.001 up to .001.

1.2.11   Highlight significant results in bold [CONTROL][SHIFT][H]

Effect: Highlights significant results in bold. This function works on a single column, containing p-values formatted as: p = 0.23.

Method: In Excel, select one column of p-values (for example, from the Wald tests output of [CONTROL][SHIFT][W]), with as many rows as desired. Execute the hotkey, significant p-values will be highlighted in bold.

 

 

1.2.12   Put slashes between results from multiple groups [CONTROL][ALT][\]

Effect: Takes results from two groups in a multiple group model, and puts them in a single column, separated by a slash.

Method: In an excel document with MPlus results obtained by running [CONTROL][SHIFT][E], put the results from the two groups next to each other (three columns per group: Label, Estimate, P-value, so six columns in total). Select the six columns and execute the hotkey. Paste the hotkey’s output manually to the desired location (usually the seventh column).

1.2.13   Take the average of slashed multiple groups output [CONTROL][ALT][A]

Effect: Takes the average of multiple group results obtained by running [CONTROL][ALT][\] to clean up the output for display in a figure. Because of standardization differences, there are often small differences between the standardized coefficients in two groups, even if parameters have been constrained to be equal. Use this function judiciously, because it oversimplifies the output. The following rule of thumb is used to determine if results will be averaged:

If the two results separated by a slash have different significance asterisks, or differ by more than .1, they are not averaged.

Method: Select a column of slash-separated results, and fire the hotkey.

1.2.14   Plug results into Visio drawing [CONTROL][ALT][V]

Effect: Copies statistics into a pre-made Visio drawing. Useful when statistics in your Visio drawing may need to be updated after re-analysis, or when making drawings for several similar models.

Method: Open an Excel document with MPlus statistics generated by 1.1.4. Open a Visio drawing with labeled shapes. In Excel, you will see 3 columns: The MPlus parameter label, the estimate (with asterisks to indicate significance), and the p-value. In the 4th column, you can now manually write labels which correspond to labels in your Visio drawing. For example, you might label an arrow in Visio with “aa”. If you now write “aa” next to a line of MPlus output in the 4th column in Excel, this statistic will replace the “aa” in the Visio drawing when you run the script. You can include as many labels as you want. Errors may occur if you repeat a label in Excel or Visio.

To run the script, select a block of results in Excel, and press [CONTROL][SHIFT][V].

This script is extremely useful, particularly when you have to run several similar models: simply annotate the Excel output for the first model, then for every subsequent model you can just select Excel cell A1, and extract the output from the next model. If you selected the same lines of MPlus output, the labels should still correspond to the correct paths in Visio, and you can quickly plug the statistics into your drawing without annotating the MPlus output again.

1.2.15   Make multi-group model [CONTROL][SHIFT][G]

Effect: Copies MODEL section from MPlus for multiple groups, and labels the parameters accordingly.

Method: Select the entire block of model syntax and fire the hotkey. A pop up asks what label should be used to identify the newly created block of syntax (e.g., B if making a block of syntax for boys). Paste the syntax below the pre-existing syntax. After doing this once, you can leave the main syntax unselected, and press CONTROL SHIFT G again to create a second group, and so on.

 

1.3      Miscellaneous hotkeys for R and other programs

1.3.1     Replace backslashes with forward slashes [CONTROL][SHIFT][\]

Effect: Replaces all backslashes with forward slashes. Useful for copy-pasting the address of a folder from windows explorer to R (which uses backslashes instead of forward slashes).

1.3.2     Turn a space or tab-separated list into a list of strings [CONTROL][SHIFT][“]

Effect: Replaces all spaces and tabs with “, “ or vice versa (spaces, not tabs). Useful for taking a list of variables from MPlus (which uses space-separated lists) to R (which uses concatenated lists of strings between quotes), or vice versa.

1.3.3     Replace all line breaks with spaces or vice versa [CONTROL][SHIFT][ENTER]

Effect: Replaces all line breaks with spaces, or vice versa. Useful for copy-pasting text from PDF to Word (PDFs usually contain redundant line breaks), or taking a list of variables separated by line breaks and turning it into a list separated by spaces.

1.3.4     Make lower case characters uppercase or vice versa [CONTROL][ALT][A]

Effect: If selected text contains uppercase letters, turn all selected text to lowercase (or vice versa). Useful to prevent problems with capitalized variable names in R (which is case sensitive).

1.3.5     Remove duplicate rows [CONTROL][ALT][D]

Effect: If selected text contains duplicate rows, remove the redundant duplicates.

1.3.6     Sort selected rows alphabetically [CONTROL][ALT][S]

Effect: Select text and press this hotkey to sort rows alphabetically (e.g., to organize rows of syntax).

1.3.7     Statistics mode [CONTROL][ALT][S]

Effect: Turns Statistics mode on, or off. When in statistics mode, a number of letters turn into statistical symbols.

m = Mean

s = SD

F = F-test

t = t-test

n = N (participants)

p = p-value ([CONTROL][P] = p < and [ALT][P] = p >)

r = r

d = Delta

a = Alpha

b = Beta

x = Chi-square

 

Method: Toggle statistics mode on or off with this hotkey.
